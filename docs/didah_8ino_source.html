55 # The Didah Voltage Monitor Appliance
56 
57 The Didah Voltage Monitor Appliance is intended to periodically report
58 battery voltage for an autonomous amatuer radio repeater station. It
59 transmits a Morse code message on the TX output at a 3.3V signal
60 level. The appliance is intended to monitor and draw power from a 12V
61 rechargable battery.
62 
63 The CPU module in Didah is a Teensy3 running at 24 MHz. There is a
64 voltage divider and voltage regulator designed support voltages up to
65 20V. The lowest voltage that can be monitored is approximately
66 2.7V.
67 
68 ## Setting Operational Parameters via Morse Code
69 
70 Two touch sensor rivets on the case labeled \c (.) and \c (-) operate
71 as an Iambic keyer. Input is entered in International Morse Code. The
72 dit duration is initially 100 milliseconds, although this can be
73 adjusted.
74 
75 Audio output is generated three ways:
76 - An internal speaker
77 - A stereo headphone jack wired to the right channel only intended
78 for demonstrations or noisy environments
79 - The TX output terminal to be wired to the repeat transmitter
80 - The TE output is an open collector that activates 120 ms prior to
81 transmission.
82 
83 A very abbreviated menu can be requested by entering an M
84 (`--`). Output can be aborted by holding a key for approximately 100
85 milliseconds, but this action may be interpreted as a command.
86 
87 Commands are keyed in three ways:
88 - Simple commands consist of a single Morse code token, such as M to
89 request the menu of commands.
90 - Commands that require arguments are entered by first entering the
91 arguments and then entering the command. This makes it possible to
92 review and correct the arguments before committing to the action.
93 - Commands that require freeform text initiate a text copying mode
94 terminated by the <span class="prosign">AR</span> prosign or by a
95 timeout.
96 
97 ## Setting Operational Parameters via Serial Console
98 
99 The USB Micro-B receptacle can be used to connect the appliance to a
100 computer. The appliance appears as a serial device (like a modem or
101 remote terminal). A terminal emulator such as Putty or the Linux
102 commands `cu` or `screen` in a command shell can be used for
103 command-line interface.
104 
105 The serial parameters are:
106 
107 Setting | Value
108 -------------|------
109 Speed | 9600 bps
110 Parity | None
111 Data | 8 bits
112 Stop bits | None
113 Flow control | None
114 
115 For Macintosh OS X, in a terminal window the following command can be used
116 to initiate a command-line dialog:
117 
118 `sudo cu -l /dev/tty.usbmodem* -s 9600 --nostop --parity=none`
119 
120 As of OS X 10.8.4, the cu program does not successfully restore the
121 serial connection when waking from sleep mode. It may be necessary to
122 kill the cu session. A cu session can usually be terminated by typing
123 
124 `~.<enter>`
125 
126 twice. Note that `<enter>` represents the Enter or Return key.
127 
128 To use screen instead of cu:
129 
130 `sudo screen /dev/tty.usbmodem* 9600`
131 
132 For Linux, in a terminal window the following command can be used
133 to initiate a command-line dialog:
134 
135 `sudo cu -l /dev/tty.usbserial-* -s 9600`
136 
137 The name of the serial device may vary from session to session. If
138 more than one serial device is connected, the wildcard would need to
139 be changed to something more specific. A little quality time with the
140 for cu or screen might yield a way to configure Didah by inserting a
141 file in the serial stream or log serial output to a file.
142 
143 ## Settings
144 
145 In the default configuration, input voltage is transmitted every 10
146 minutes in Morse Code as follows:
147 
148 `BAT 4801 MV ? 4801 MV`
149 
150 The message content, repetition interval, transmission rate, pitch
151 (side tone), and alarm level can all be configured. The configuration can be
152 saved to persistent storage.
153 
154 ## Command-line Interaction
155 
156 Command | Description
157 --------|------------
158  ? | Help
159  A | Set low-voltage alarm threshold in millivolts [2700-16000]
160  B | Set high-voltage alarm threshold in millivolts [2700-16000]
161  D | Set dit duration in milliseconds [20-200]
162  F | Set normal announcement format
163  G | Set low-voltage announcement format
164  H | Set high-voltage announcement format
165  I | Set normal announcement interval in seconds [20-86400]
166  J | Set alarm announcement interval in seconds [20-86400]
167  L | Print settings and log
168  O | Load settings from EEPROM
169  P | Set output frequency (pitch) in Hz [55-1760]
170  S | Save settings to EEPROM
171  T | Set the 24-hour time [0000-2359]
172 
173 ### The Help Command
174 
175 The Help command (`?`) prints a banner, settings, and a brief list of commands.
176 
177 ### Setting the Low-voltage Alarm Threshold
178 
179 The Low-voltage Alarm Threshold command (`A`) prompts for a value in
180 millivolts, ranging from 2700 to 16000 millivolts. When the input voltage
181 falls below this threshold, alternate values are used for the announcement
182 repetition format and interval. The red LED is lit continuously at a dim level.
183 The default threshold is 2700 millivolts.
184 \htmlonly
185 <p><code>
186 &gt; <b>A</b></br>
187 Enter low-voltage threshold in mV [2700-16000]: <b>4000</b></br>
188 Low-voltage threshold: 4000 mV</br>
189 &gt;
190 </code></p>
191 \endhtmlonly
192 
193 ### Setting the Low-voltage Alarm Threshold
194 
195 The High-voltage Alarm Threshold command (`B`) prompts for a value in
196 millivolts, ranging from 2700 to 16000 millivolts. When the input voltage
197 rises above this threshold, alternate values are used for the announcement
198 repetition format and interval. The red LED is lit continuously at a dim level.
199 The default threshold is 16000 millivolts.
200 
201 ### Setting the Dit Duration
202 
203 The Dit Duration command (`D`) prompts for a value in milliseconds,
204 ranging from 20 to 200. This sets the rate at which Morse Code is
205 entered and transmitted. The default duration is 40 milliseconds.
206 
207 ### Setting the Normal Announcement Format
208 
209 The Normal Announcement Format command (`F`) prompts the user for a text
210 template for periodic announcements when the voltage is at a nominal
211 level.
212 
213 The format template for announcements uses the following notation:
214 
215 Symbol | Interpretation
216 ---------------|---------------
217 A-Z | Literal character
218 0-9 | Literal number
219 \e space | Pause for one dit unit
220 \e punctuation | Literal punctuation: !"$&'()+,-./:;=?@_
221 \e ^ps^ | Prosign of up to seven dits and dahs. \n For example prosign BK is transmitted \c -....-.
222 \%V | Monitor voltage in millivolts
223 \%F | Temperature in degrees Fahrenheit
224 \%C | Temperature in degrees Celsius
225 \%T | Time of day in four-digit 24-hour time: HHMM
226 \%U | Uptime in four digits: HHMM
227 
228 The default announcement format is `BAT 4801 MV ? 4801 MV`.
229 
230 ### Setting the Low-Voltage Alarm Announcement Format
231 
232 The Low-Voltage Announcement Format command (`G`) prompts the user for
233 a text template for periodic announcements when the voltage is below
234 the nominal range. The template format is identical to the nominal
235 announcement format above. The default alarm format is `BAT 4801 MV ?
236 4801 MV`.
237 
238 ### Setting the High-Voltage Alarm Announcement Format
239 
240 The High-Voltage Alarm Format command (`H`) prompts the user for a
241 text template for periodic announcements when the voltage is above the
242 nominal ramge. The template format is identical to the nominal
243 announcement format above. The default alarm format is `BAT 16000 MV ?
244 16000 MV`.
245 
246 ### Setting the Normal Announcement Interval
247 
248 The Normal Announcement Interval command (`I`) prompts the user for a
249 time interval in seconds ranging from 20 to 86,400 (24 hours). When
250 the voltage is nominal, and the time in seconds is a multiple of this
251 value, a nominal announcement is transmitted. The default announcement
252 interval is 600 seconds (ten minutes).
253 
254 ### Setting the Alarm Announcement Interval
255 
256 The Alarm Announcement Interval command (`J`) prompts the user for a
257 time interval in seconds ranging from 20 to 86,400 (24 hours). When
258 the voltage is at or below the alarm threshold, and the time in
259 seconds is a multiple of this value, an alarm announcement is
260 transmitted. The default alarm interval is 30 seconds.
261

295 ### Loading Persistent Settings
296 
297 The Load Settings command (O) restores settings saved previously. \b
298 N.B. This command takes effect immediately without confirmation. Any
299 unsaved changes are overwritten.
300 
301 In the event of a temporary power failure, settings revert to their
302 default values; the settings saved by the user are not automatically
303 restored.
304 
305 ### Setting the Output Frequency (Side Tone)
306 
307 The output frequency or side tone can be set using the (P) command. The
308 frequency range is 55 to 1760 Hz. The default frequency is 750 Hz.
309 
310 ### Saving Settings
311 
312 The Save command (S) stores the current settings to EEPROM. These settings
313 are preserved when there is no power to the appliance. \b N.B. This
314 command takes effect immediately without confirmation. Any unsaved
315 changes are overwritten.
316 
317 ### Setting the Time
318 
319 The Time command (T) prompts the user for the current 24-hour time
320 [0000-2359]. The time in seconds is used to determine the timing of
321 the announcements. \b N.B. The uptime stored in EEPROM does not get
322 updated unless the time is set. This allows the user to see how long
323 the appliance was up before a power failure.

325 ## Morse Code Interaction
326 
327 The Morse Code interface is designed to provide a quick method for an
328 experienced CW operator to adjust the settings of the voltage monitor
329 appliance. As mentioned above, commands are keyed in three ways:
330 - Simple commands consist of a single Morse code token, such as M to
331 request the menu of commands.
332 - Commands that require arguments are entered after the arguments so
333 that it is possible to review and correct the arguments before
334 entering the command.
335 - Commands that require freeform text initiate a text copying mode
336 terminated by the <span class="prosign">AR</span> prosign or by a
337 timeout.
338 
339 ### The Menu Command
340 
341 The Menu Command (`--`) transmits an abbreviated list of commands:
342 ~~~~
343 A LOW V IN MV,
344 B HIGH V IN MV,
345 C CLEAR,
346 D DIT MS,
347 F MSG FMT,
348 G ALARM LOW FMT,
349 H ALARM HIGH FMT,
350 I RPT SECS,
351 J RPT ALARM SECS,
352 L LIST,
353 M MENU,
354 O LOAD,
355 P TONE HZ,
356 S SAVE,
357 T TIME,
358 U UPTIME.
359 K~~~~

361 ### Setting the Low-voltage Alarm Threshold
362 
363 The `.-` (A) command expects a number ranging from 2700 to 16000 to
364 have been entered and sets the low-voltage alarm threshold
365 accordingly.
366 
367 For example, to set the low-voltage alarm threshold to 5V,
368 
369 User: `...../-----/-----/-----/.-` \n
370 System: `-.-` \n
371 
372 If the number is out of range, the system will transmit 8 dits
373 followed by the word "RANGE".
374 
375 ### Setting the High-voltage Alarm Threshold
376 
377 The `-...` (B) command expects a number ranging from 2700 to 16000 to
378 have been entered and sets the high-voltage alarm threshold
379 accordingly.
380 
381 For example, to set the high-voltage alarm threshold to 14V,
382 
383 User: `.----/....-/-----/-----/-----/-...` \n
384 System: `-.-` \n
385 
386 If the number is out of range, the system will transmit 8 dits
387 followed by the word "RANGE".
388 
389 ### Setting the Dit Duration
390 
391 The `-..` (D) command expects a number of milliseconds ranging from 20
392 to 200 to have been entered and sets the dit duration accordingly.
393 
394 For example, to set the dit duration to 50ms,
395 
396 User: `...../-----/.-` \n
397 System: `-.-` \n
398 
399 If the number is out of range, the system will transmit 8 dits
400 followed by the word "RANGE".
401 
402 ### Setting the Normal Announcement Format
403 
404 The `..-.` (F) command causes the system to prompt for a format template.
405 The system responds with
406 `TX MSG END WITH` <span class="prosign">AR</span> `K`.
407 
408 For example, here is how to make the system transmit the voltage
409 followed by MV:
410 
411 User: `.._.` \n
412 System: `_ _.._ /__ ... __. /. _. _.. /.__ .. _ .... /._._. //_._` \n
413 User: `..--../...-//--/...-/.-.-.` \n
414 System: `-.-` \n
415 
416 Or in text,
417 
418 User: F\n
419 System: TX MSG END WITH <span class="prosign">AR</span> OVER\n
420 User: ?V MV <span class="prosign">AR</span>\n
421 System: OVER\n
422 
423 \b N.B. To indicate a placeholder, key a question mark (`..--..`)
424 followed immediately by one of V, F, C, T, or U. A question mark
425 followed by a pause or any other token is transmitted literally.
426 
427 The format template is used for generating periodic messages when the
428 voltage level is nominal just prior to message generation. The default
429 format is `BAT 4801 MV ? 4801 MV`.
430 
431 ### Setting the Alarm Low-Voltage Announcement Format
432 
433 The `--.` (G) command causes the system to prompt for a low-voltage
434 alert format string. Refer to the F command above for details.
435 
436 ### Setting the Alarm High-Voltage Announcement Format
437 
438 The `....` (H) command causes the system to prompt for a high-voltage
439 alert format string. Refer to the F command above for details.
440 
441 ### Setting the Normal Announcement Interval
442 
443 The `..` (I) command expects a number of seconds ranging from 20
444 to 86400 to have been entered and sets the nominal announcement interval
445 accordingly. The interval is coordinated with the time of day. The default
446 interval is 600 seconds (ten minutes).
447 
448 ### Setting the Alarm Announcement Interval
449 
450 The `.---` (J) command expects a number of seconds ranging from 20
451 to 86400 to have been entered and sets the alarm announcement interval
452 accordingly. The interval is coordinated with the time of day. The default
453 alarm interval is 30 seconds.
454 
455 ### The Log Command
456 
457 The `.-..` (L) command transmits a log of the current settings.
458 For example,
459 
460 ~~~~
461 SETTINGS.
462 MSG RPT SKED IS 600 SECONDS.
463 MSG FORMAT IS BAT ?V MV ? ?V MV .
464 ALARM VOLTAGE IS 4000 MV.
465 ALARM MSG RPT SKED IS 30 SECONDS.
466 ALARM MSG FORMAT IS BAT ?V MV ? ?V MV .~~~~
467 
468 ### Loading Persistent Settings
469 
470 The `---` (O) command loads settings from EEPROM. Saved settings are not
471 automatically restored at power on.
472 
473 ### Setting the Output Frequency (Side Tone)
474 
475 The `.--.` (P) command sets the output frequency or side tone. The
476 frequency range is 55 to 1760 Hz. The default frequency is 750.
477 
478 ### Saving Settings
479 
480 The `...` (S) command saves settings to EEPROM. Settings are retained
481 when there is no power. Settings are not automatically loaded when
482 power is restored, thus providing a fail-safe mechanism for recovering
483 from mis-configuration.
484 
485 ### Setting the Time
486 
487 The `-` (T) command expects the 24-hour time to have been entered as
488 four digits HHMM (leading zeros may be omitted) and sets the time
489 accordingly. The seconds are implicitly set to zero. Until the time
490 is set, the uptime stored in EEPROM is not updated.
491 
492 ### Stack Manipulation Commands
493 
494 The `-.-.` (C) command clears the stack.
495 
496 The `......` (<span class="prosign">SS</span>) command pops the stack
497 (erases the last element).
498 
499 The `-..-` (X) command exchanges the top two entries of the stack.
