53 /** \page 2 Instructions
54 # The Didah Voltage Monitor Appliance
55 
56 The Didah Voltage Monitor Appliance is intended to periodically report
57 battery voltage for an autonomous amatuer radio repeater station. It
58 transmits a Morse code message on the TX output at a 3.3V signal
59 level. The appliance is intended to monitor and draw power from a 12V
60 rechargable battery.
61 
62 The CPU module in Didah is a Teensy3 running at 24 MHz. There is a
63 voltage divider and voltage regulator designed support voltages up to
64 20V. The lowest voltage that can be monitored is approximately
65 2.7V.
66 
67 ## Setting Operational Parameters via Morse Code
68 
69 Two touch sensor rivets on the case labeled \c (.) and \c (-) operate
70 as an Iambic keyer. Input is entered in International Morse Code. The
71 dit duration is initially 100 milliseconds, although this can be
72 adjusted.
73 
74 Audio output is generated three ways:
75 - An internal speaker
76 - A stereo headphone jack wired to the right channel only intended
77 for demonstrations or noisy environments
78 - The TX output terminal to be wired to the repeat transmitter
79 - The TE output is an open collector that activates 120 ms prior to
80 transmission.
81 
82 A very abbreviated menu can be requested by entering an M
83 (`--`). Output can be aborted by holding a key for approximately 100
84 milliseconds, but this action may be interpreted as a command.
85 
86 Commands are keyed in three ways:
87 - Simple commands consist of a single Morse code token, such as M to
88 request the menu of commands.
89 - Commands that require arguments are entered by first entering the
90 arguments and then entering the command. This makes it possible to
91 review and correct the arguments before committing to the action.
92 - Commands that require freeform text initiate a text copying mode
93 terminated by the <span class="prosign">AR</span> prosign or by a
94 timeout.
95 
96 ## Setting Operational Parameters via Serial Console
97 
98 The USB Micro-B receptacle can be used to connect the appliance to a
99 computer. The appliance appears as a serial device (like a modem or
100 remote terminal). A terminal emulator such as Putty or the Linux
101 commands `cu` or `screen` in a command shell can be used for
102 command-line interface.
103 
104 The serial parameters are:
105 
106 Setting | Value
107 -------------|------
108 Speed | 9600 bps
109 Parity | None
110 Data | 8 bits
111 Stop bits | None
112 Flow control | None
113 
114 For Macintosh OS X, in a terminal window the following command can be used
115 to initiate a command-line dialog:
116 
117 `sudo cu -l /dev/tty.usbmodem* -s 9600 --nostop --parity=none`
118 
119 As of OS X 10.8.4, the cu program does not successfully restore the
120 serial connection when waking from sleep mode. It may be necessary to
121 kill the cu session. A cu session can usually be terminated by typing
122 
123 `~.<enter>`
124 
125 twice. Note that `<enter>` represents the Enter or Return key.
126 
127 To use screen instead of cu:
128 
129 `sudo screen /dev/tty.usbmodem* 9600`
130 
131 For Linux, in a terminal window the following command can be used
132 to initiate a command-line dialog:
133 
134 `sudo cu -l /dev/tty.usbserial-* -s 9600`
135 
136 The name of the serial device may vary from session to session. If
137 more than one serial device is connected, the wildcard would need to
138 be changed to something more specific. A little quality time with the
139 for cu or screen might yield a way to configure Didah by inserting a
140 file in the serial stream or log serial output to a file.
141 
142 ## Settings
143 
144 In the default configuration, input voltage is transmitted every 10
145 minutes in Morse Code as follows:
146 
147 `BAT 4801 MV ? 4801 MV`
148 
149 The message content, repetition interval, transmission rate, pitch
150 (side tone), and alarm level can all be configured. The configuration can be
151 saved to persistent storage.
152 
153 ## Command-line Interaction
154 
155 Command | Description
156 --------|------------
157  ? | Help
158  A | Set low-voltage alarm threshold in millivolts [2700-16000]
159  B | Set high-voltage alarm threshold in millivolts [2700-16000]
160  D | Set dit duration in milliseconds [20-200]
161  F | Set normal announcement format
162  G | Set low-voltage announcement format
163  H | Set high-voltage announcement format
164  I | Set normal announcement interval in seconds [20-86400]
165  J | Set alarm announcement interval in seconds [20-86400]
166  L | Print settings and log
167  O | Load settings from EEPROM
168  P | Set output frequency (pitch) in Hz [55-1760]
169  S | Save settings to EEPROM
170  T | Set the 24-hour time [0000-2359]
171 
172 ### The Help Command
173 
174 The Help command (`?`) prints a banner, settings, and a brief list of commands.
175 
176 ### Setting the Low-voltage Alarm Threshold
177 
178 The Low-voltage Alarm Threshold command (`A`) prompts for a value in
179 millivolts, ranging from 2700 to 16000 millivolts. When the input voltage
180 falls below this threshold, alternate values are used for the announcement
181 repetition format and interval. The red LED is lit continuously at a dim level.
182 The default threshold is 2700 millivolts.
183 \htmlonly
184 <p><code>
185 &gt; <b>A</b></br>
186 Enter low-voltage threshold in mV [2700-16000]: <b>4000</b></br>
187 Low-voltage threshold: 4000 mV</br>
188 &gt;
189 </code></p>
190 \endhtmlonly
191 
192 ### Setting the Low-voltage Alarm Threshold
193 
194 The High-voltage Alarm Threshold command (`B`) prompts for a value in
195 millivolts, ranging from 2700 to 16000 millivolts. When the input voltage
196 rises above this threshold, alternate values are used for the announcement
197 repetition format and interval. The red LED is lit continuously at a dim level.
198 The default threshold is 16000 millivolts.
199 
200 ### Setting the Dit Duration
201 
202 The Dit Duration command (`D`) prompts for a value in milliseconds,
203 ranging from 20 to 200. This sets the rate at which Morse Code is
204 entered and transmitted. The default duration is 40 milliseconds.
205 
206 ### Setting the Normal Announcement Format
207 
208 The Normal Announcement Format command (`F`) prompts the user for a text
209 template for periodic announcements when the voltage is at a nominal
210 level.
211 
212 The format template for announcements uses the following notation:
213 
214 Symbol | Interpretation
215 ---------------|---------------
216 A-Z | Literal character
217 0-9 | Literal number
218 \e space | Pause for one dit unit
219 \e punctuation | Literal punctuation: !"$&'()+,-./:;=?@_
220 \e ^ps^ | Prosign of up to seven dits and dahs. \n For example prosign BK is transmitted \c -....-.
221 \%V | Monitor voltage in millivolts
222 \%F | Temperature in degrees Fahrenheit
223 \%C | Temperature in degrees Celsius
224 \%T | Time of day in four-digit 24-hour time: HHMM
225 \%U | Uptime in four digits: HHMM
226 
227 The default announcement format is `BAT 4801 MV ? 4801 MV`.
228 
229 ### Setting the Low-Voltage Alarm Announcement Format
230 
231 The Low-Voltage Announcement Format command (`G`) prompts the user for
232 a text template for periodic announcements when the voltage is below
233 the nominal range. The template format is identical to the nominal
234 announcement format above. The default alarm format is `BAT 4801 MV ?
235 4801 MV`.
236 
237 ### Setting the High-Voltage Alarm Announcement Format
238 
239 The High-Voltage Alarm Format command (`H`) prompts the user for a
240 text template for periodic announcements when the voltage is above the
241 nominal ramge. The template format is identical to the nominal
242 announcement format above. The default alarm format is `BAT 16000 MV ?
243 16000 MV`.
244 
245 ### Setting the Normal Announcement Interval
246 
247 The Normal Announcement Interval command (`I`) prompts the user for a
248 time interval in seconds ranging from 20 to 86,400 (24 hours). When
249 the voltage is nominal, and the time in seconds is a multiple of this
250 value, a nominal announcement is transmitted. The default announcement
251 interval is 600 seconds (ten minutes).
252 
253 ### Setting the Alarm Announcement Interval
254 
255 The Alarm Announcement Interval command (`J`) prompts the user for a
256 time interval in seconds ranging from 20 to 86,400 (24 hours). When
257 the voltage is at or below the alarm threshold, and the time in
258 seconds is a multiple of this value, an alarm announcement is
259 transmitted. The default alarm interval is 30 seconds.
260

294 ### Loading Persistent Settings
295 
296 The Load Settings command (O) restores settings saved previously. \b
297 N.B. This command takes effect immediately without confirmation. Any
298 unsaved changes are overwritten.
299 
300 In the event of a temporary power failure, settings revert to their
301 default values; the settings saved by the user are not automatically
302 restored.
303 
304 ### Setting the Output Frequency (Side Tone)
305 
306 The output frequency or side tone can be set using the (P) command. The
307 frequency range is 55 to 1760 Hz. The default frequency is 750 Hz.
308 
309 ### Saving Settings
310 
311 The Save command (S) stores the current settings to EEPROM. These settings
312 are preserved when there is no power to the appliance. \b N.B. This
313 command takes effect immediately without confirmation. Any unsaved
314 changes are overwritten.
315 
316 ### Setting the Time
317 
318 The Time command (T) prompts the user for the current 24-hour time
319 [0000-2359]. The time in seconds is used to determine the timing of
320 the announcements. \b N.B. The uptime stored in EEPROM does not get
321 updated unless the time is set. This allows the user to see how long
322 the appliance was up before a power failure.
323 
324 ## Morse Code Interaction
325 
326 The Morse Code interface is designed to provide a quick method for an
327 experienced CW operator to adjust the settings of the voltage monitor
328 appliance. As mentioned above, commands are keyed in three ways:
329 - Simple commands consist of a single Morse code token, such as M to
330 request the menu of commands.
331 - Commands that require arguments are entered after the arguments so
332 that it is possible to review and correct the arguments before
333 entering the command.
334 - Commands that require freeform text initiate a text copying mode
335 terminated by the <span class="prosign">AR</span> prosign or by a
336 timeout.
337 
338 ### The Menu Command
339 
340 The Menu Command (`--`) transmits an abbreviated list of commands:
341 ~~~~
342 A LOW V IN MV,
343 B HIGH V IN MV,
344 C CLEAR,
345 D DIT MS,
346 F MSG FMT,
347 G ALARM LOW FMT,
348 H ALARM HIGH FMT,
349 I RPT SECS,
350 J RPT ALARM SECS,
351 L LIST,
352 M MENU,
353 O LOAD,
354 P TONE HZ,
355 S SAVE,
356 T TIME,
357 U UPTIME.
358 K~~~~

360 ### Setting the Low-voltage Alarm Threshold
361 
362 The `.-` (A) command expects a number ranging from 2700 to 16000 to
363 have been entered and sets the low-voltage alarm threshold
364 accordingly.
365 
366 For example, to set the low-voltage alarm threshold to 5V,
367 
368 User: `...../-----/-----/-----/.-` \n
369 System: `-.-` \n
370 
371 If the number is out of range, the system will transmit 8 dits
372 followed by the word "RANGE".
373 
374 ### Setting the High-voltage Alarm Threshold
375 
376 The `-...` (B) command expects a number ranging from 2700 to 16000 to
377 have been entered and sets the high-voltage alarm threshold
378 accordingly.
379 
380 For example, to set the high-voltage alarm threshold to 14V,
381 
382 User: `.----/....-/-----/-----/-----/-...` \n
383 System: `-.-` \n
384 
385 If the number is out of range, the system will transmit 8 dits
386 followed by the word "RANGE".
387 
388 ### Setting the Dit Duration
389 
390 The `-..` (D) command expects a number of milliseconds ranging from 20
391 to 200 to have been entered and sets the dit duration accordingly.
392 
393 For example, to set the dit duration to 50ms,
394 
395 User: `...../-----/.-` \n
396 System: `-.-` \n
397 
398 If the number is out of range, the system will transmit 8 dits
399 followed by the word "RANGE".
400 
401 ### Setting the Normal Announcement Format
402 
403 The `..-.` (F) command causes the system to prompt for a format template.
404 The system responds with
405 `TX MSG END WITH` <span class="prosign">AR</span> `K`.
406 
407 For example, here is how to make the system transmit the voltage
408 followed by MV:
409 
410 User: `.._.` \n
411 System: `_ _.._ /__ ... __. /. _. _.. /.__ .. _ .... /._._. //_._` \n
412 User: `..--../...-//--/...-/.-.-.` \n
413 System: `-.-` \n
414 
415 Or in text,
416 
417 User: F\n
418 System: TX MSG END WITH <span class="prosign">AR</span> OVER\n
419 User: ?V MV <span class="prosign">AR</span>\n
420 System: OVER\n
421 
422 \b N.B. To indicate a placeholder, key a question mark (`..--..`)
423 followed immediately by one of V, F, C, T, or U. A question mark
424 followed by a pause or any other token is transmitted literally.
425 
426 The format template is used for generating periodic messages when the
427 voltage level is nominal just prior to message generation. The default
428 format is `BAT 4801 MV ? 4801 MV`.
429 
430 ### Setting the Alarm Low-Voltage Announcement Format
431 
432 The `--.` (G) command causes the system to prompt for a low-voltage
433 alert format string. Refer to the F command above for details.
434 
435 ### Setting the Alarm High-Voltage Announcement Format
436 
437 The `....` (H) command causes the system to prompt for a high-voltage
438 alert format string. Refer to the F command above for details.
439 
440 ### Setting the Normal Announcement Interval
441 
442 The `..` (I) command expects a number of seconds ranging from 20
443 to 86400 to have been entered and sets the nominal announcement interval
444 accordingly. The interval is coordinated with the time of day. The default
445 interval is 600 seconds (ten minutes).
446 
447 ### Setting the Alarm Announcement Interval
448 
449 The `.---` (J) command expects a number of seconds ranging from 20
450 to 86400 to have been entered and sets the alarm announcement interval
451 accordingly. The interval is coordinated with the time of day. The default
452 alarm interval is 30 seconds.
453 
454 ### The Log Command
455 
456 The `.-..` (L) command transmits a log of the current settings.
457 For example,
458 
459 ~~~~
460 SETTINGS.
461 MSG RPT SKED IS 600 SECONDS.
462 MSG FORMAT IS BAT ?V MV ? ?V MV .
463 ALARM VOLTAGE IS 4000 MV.
464 ALARM MSG RPT SKED IS 30 SECONDS.
465 ALARM MSG FORMAT IS BAT ?V MV ? ?V MV .~~~~
466 
467 ### Loading Persistent Settings
468 
469 The `---` (O) command loads settings from EEPROM. Saved settings are not
470 automatically restored at power on.
471 
472 ### Setting the Output Frequency (Side Tone)
473 
474 The `.--.` (P) command sets the output frequency or side tone. The
475 frequency range is 55 to 1760 Hz. The default frequency is 750.
476 
477 ### Saving Settings
478 
479 The `...` (S) command saves settings to EEPROM. Settings are retained
480 when there is no power. Settings are not automatically loaded when
481 power is restored, thus providing a fail-safe mechanism for recovering
482 from mis-configuration.
483 
484 ### Setting the Time
485 
486 The `-` (T) command expects the 24-hour time to have been entered as
487 four digits HHMM (leading zeros may be omitted) and sets the time
488 accordingly. The seconds are implicitly set to zero. Until the time
489 is set, the uptime stored in EEPROM is not updated.
490 
491 ### Stack Manipulation Commands
492 
493 The `-.-.` (C) command clears the stack.
494 
495 The `......` (<span class="prosign">SS</span>) command pops the stack
496 (erases the last element).
497 
498 The `-..-` (X) command exchanges the top two entries of the stack.
